# Pi — Quick Reference

This file is auto-loaded as context in **every pi session**. It is a running map
of what is available to you (the agent) and to the user. Sections marked
`auto` are regenerated by the `pi-reference` extension on every session start —
do not hand-edit them.

> **Junior dev start here:** jump to ["I want to..."](#i-want-to) below.

## Editor quick reference

| Keys | Action |
|---|---|
| `Enter` | Send message (or queue a *steering* message while streaming) |
| `Alt+Enter` | Queue a *follow-up* message (delivered after agent finishes all work) |
| `Shift+Enter` | New line |
| `Esc` | Cancel current turn / abort |
| `Esc Esc` | Open `/tree` |
| `Ctrl+C` | Clear editor; twice to quit |
| `Ctrl+L` | Model selector |
| `Ctrl+P` / `Shift+Ctrl+P` | Cycle scoped models forward/back |
| `Shift+Tab` | Cycle thinking level |
| `Ctrl+O` | Collapse/expand tool output |
| `Ctrl+T` | Collapse/expand thinking blocks |
| `Ctrl+V` (`Alt+V` on Windows) | Paste image |
| `Ctrl+G` | Open `$EDITOR` for the current draft |
| `@` | Fuzzy-find a file to reference |
| `Tab` | Path completion |
| `!cmd` | Run bash, send output to model |
| `!!cmd` | Run bash, do NOT send output |

Full list: `/hotkeys`. Customize via `~/.pi/agent/keybindings.json`.

## Built-in tools available to the agent

These are what the model can call without any extensions.

| Tool | Purpose |
|---|---|
| `read` | Read a file (text or image), with offset/limit |
| `write` | Create or overwrite a file |
| `edit` | Exact-text replacement edits in a file |
| `bash` | Run a shell command (with timeout) |
| `grep` | Search file contents |
| `find` | Locate files by name/glob |
| `ls` | List a directory |

Restrict tools with `--tools read,grep,find,ls -p` (read-only print mode), or
`--no-builtin-tools` to disable defaults while keeping extension tools.

## Lifecycle hooks (for extension authors)

When you build a pi extension, you can subscribe to these events. Grouped by
when they fire. Full docs: `/usr/local/lib/node_modules/@earendil-works/pi-coding-agent/docs/extensions.md`.

### Startup / shutdown
- `session_start` — session loaded (reason: `startup` | `reload` | `new` | `resume` | `fork`).
- `resources_discover` — contribute extra skill/prompt/theme paths.
- `session_before_switch` — about to `/new` or `/resume`; can cancel.
- `session_before_fork` — about to `/fork` or `/clone`; can cancel.
- `session_shutdown` — runtime tearing down; do cleanup here.

### User input
- `input` — raw user text, before skill/template expansion. Can transform or handle.
- `user_bash` — user ran `!cmd` or `!!cmd`. Can intercept (e.g. SSH).

### Agent turn
- `before_agent_start` — inject a message OR modify the system prompt for this turn.
- `agent_start` / `agent_end` — one user prompt's full agent loop.
- `turn_start` / `turn_end` — one LLM response + its tool calls.
- `message_start` / `message_update` / `message_end` — message lifecycle (assistant, user, toolResult). `message_end` can replace the finalized message.
- `context` — mutate messages right before each LLM call.
- `before_provider_request` — inspect/replace the provider payload.
- `after_provider_response` — read response status/headers (e.g. handle 429).

### Tool calls
- `tool_execution_start` — preflight, before the tool runs.
- `tool_call` — can BLOCK (`return { block: true, reason }`) or mutate `event.input` in place.
- `tool_execution_update` — streaming progress.
- `tool_result` — can MODIFY the result (chained middleware).
- `tool_execution_end` — tool finished.

### Compaction / tree
- `session_before_compact` / `session_compact` — customize or cancel compaction.
- `session_before_tree` / `session_tree` — customize or cancel `/tree`.

### Model
- `model_select` — model changed (source: `set` | `cycle` | `restore`).
- `thinking_level_select` — thinking level changed.

## Filesystem layout cheatsheet

```
~/.pi/agent/
├── AGENTS.md            ← this file (global context, auto-loaded everywhere)
├── settings.json        ← global settings
├── keybindings.json     ← custom keybindings
├── auth.json            ← OAuth tokens (do not commit)
├── prompts/             ← /name prompt templates
├── skills/<name>/SKILL.md  ← /skill:name on-demand capabilities
├── chains/*.chain.md    ← multi-step chains
├── extensions/<name>/index.ts  ← TS extensions
├── themes/              ← color themes
└── sessions/            ← session JSONL files

<project>/
├── AGENTS.md            ← project-local context (loaded after global)
├── .pi/
│   ├── settings.json    ← project settings (overrides global)
│   ├── SYSTEM.md        ← replace system prompt (project)
│   ├── APPEND_SYSTEM.md ← append to system prompt
│   ├── prompts/, skills/, chains/, extensions/, themes/
└── UML.md               ← architectural memory (managed by uml-tracker)
```

## CLI flags worth remembering

```
pi                          # interactive, picks up latest session if -c
pi -c                       # continue most recent
pi -r                       # browse past sessions
pi --no-session             # ephemeral, don't save
pi -p "prompt"              # one-shot print mode
pi --mode json              # event stream as JSON lines
pi --model sonnet:high      # model + thinking level shorthand
pi --provider openai --model gpt-4o
pi --tools read,grep,find,ls -p "Review the code"   # read-only
pi --thinking high "..."
pi -e ./my-ext.ts           # load an extension ad-hoc (testing)
pi @file1.ts @file2.ts "..."  # include files in the prompt
cat README.md | pi -p "Summarize"   # piped stdin merges into prompt
```
